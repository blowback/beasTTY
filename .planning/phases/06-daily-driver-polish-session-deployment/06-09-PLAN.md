<objective>
Close the chrome-buttons-non-functional blocker reported in 06-UAT.md Test 1
(blocking Tests 2-7).

Root cause (per .planning/debug/phase-6-chrome-buttons-non-functional.md):
Phase 6 Plan 06-06 introduced a `prefs.subscribe(applyPrefs)` pattern. After
every 250 ms debounced `savePrefs`, `flushPrefs()` fires all subscribers
including `applyPrefs`, which unconditionally re-mutates DOM form controls
from the cached prefs blob. This races against any code path that mutates
the DOM directly without going through savePrefs — the proven instance is
Phase 5's `snapPreset()` in serial.js:302-310, which sets serialEls.*.value
to the MicroBeast preset without syncing the cached prefs blob. 250 ms later,
applyPrefs re-applies the stale cached values, undoing snapPreset.

The user-reported "click does nothing" symptom is consistent with the same
race (visible click effect reverts ~250 ms later, looks like the click
never landed).

Two-part fix, both must ship:

1. MINIMAL FIX (closes the proven race) — snapPreset() also calls
   savePrefsFn({ serial: PRESET_BLOB }) so the cached prefs blob stays
   in sync with the visible form, making the next applyPrefs idempotent.

2. STRUCTURAL FIX (closes the entire race surface) — flushPrefs() in
   prefs.js no longer fires subscribers. Only resetPrefs() does. Rationale:
   when savePrefs is triggered by the user's own DOM-mutating click, the
   DOM is already at the saved value; re-applying it is at best a no-op
   and at worst a race. Subscribers belong on the external/reset/import
   path, NOT the routine debounced-save path.

3. REGRESSION TESTS — extend prefs.spec.js with a no-revert assertion;
   confirm config.spec.js#Reset preset now runs green without retries.

Purpose: Restore working top-bar chrome (Connect, Clear, theme toggle,
phosphor radios) so the user can complete UAT Tests 1-7. Out of scope:
WR-01..WR-05 advisory items (separate /gsd-code-review-fix run); rewriting
the prefs blob shape; adding new prefs.

Output: Two production-code edits, two test files updated/extended, all
existing Playwright suites still green, the previously-flaky
'Reset to MicroBeast preset' transport test now deterministic.
