<objective>
Fix Gap 1 (reload hang / "Page unresponsive" dialog — UAT Test 3, blocker).

**Purpose:**
The beforeunload handler at www/transport/serial.js:105-111 calls reader.cancel() + port.close() but never calls reader.releaseLock() nor writer.releaseLock(). Per the WHATWG/MDN contract for SerialPort.close(), the returned promise only resolves when BOTH port.readable and port.writable are unlocked. The handler's close() therefore never resolves, stalling Chromium's renderer tear-down on the OLD page — which Chromium surfaces as "Page unresponsive" (clicking Wait doesn't help because the contract CANNOT be satisfied without releaseLock calls).

This plan closes the blocker by making the beforeunload handler mirror the teardown() helper's lock hygiene (but synchronously, to respect the beforeunload time budget), and by adding a `shuttingDown` module-scope flag so runReadLoop's outer `while (p.readable)` does not re-acquire a fresh reader after the cancel-break.

**Output:**
- Corrected beforeunload handler in www/transport/serial.js that releases the reader and writer locks before calling port.close().
- `shuttingDown` guard in runReadLoop so no fresh reader is re-acquired during tear-down.
- Updated leading comment block reflecting that cancel() != releaseLock() and that close() requires unlocked streams.
- Playwright regression spec (www/tests/transport/lifecycle.spec.js) asserting release-before-close ordering via an instrumented mock port.
- Minimal mock-serial.js instrumentation (window.__mockLockLog) so the spec can introspect lock calls.
</objective>
