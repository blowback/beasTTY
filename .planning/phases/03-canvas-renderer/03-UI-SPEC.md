# Phase 3 — UI Design Contract

> Visual and interaction contract for the Canvas Renderer phase. Generated by
> gsd-ui-researcher; verified by gsd-ui-checker.
>
> **Surface area note:** This phase's "UI" is a terminal emulator — a canvas
> rendering surface plus a minimal top-bar chrome. Nearly every contract field
> below was pre-locked in `03-CONTEXT.md` (decisions D-01..D-15). This
> document translates those decisions into a single tabular design contract
> the planner and executor can point at as visual source of truth.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (framework-free vanilla ES modules + hand-authored CSS — PROJECT.md and Phase 2 D-14 lock this posture; shadcn/Tailwind intentionally excluded) |
| Preset | not applicable |
| Component library | none (no React/Vue/Svelte; DOM APIs directly) |
| Icon library | none (text-only chrome; theme toggle + phosphor radio-group use text labels, not icons) |
| Font — CRT theme | Hand-drawn 8×16 public-domain bitmap ROM font shipped as `Uint8Array` in `www/renderer/bitmap-font.js` (D-01). Reference: IBM VGA 8×16 ROM (MIT-compatible public domain). Executor finalises exact ROM selection during planning. |
| Font — Clean theme | JetBrains Mono Regular (400), self-hosted subset WOFF2 at `www/assets/fonts/jetbrains-mono-regular.woff2`, `@font-face` with `font-display: block` (D-02). Fallback chain: `ui-monospace, SFMono-Regular, Menlo, Consolas, monospace`. **`block` not `swap`** — ROADMAP SC-1 requires no font-fallback flash on first paint. |
| Font — Chrome (top bar, Debug details) | Clean-theme fallback chain only: `ui-monospace, SFMono-Regular, Menlo, Consolas, monospace`. Chrome never uses the CRT bitmap font or JetBrains Mono; this keeps chrome rendering theme-independent and avoids a second WOFF2 subset. |

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Top-bar internal gaps (between label and button, between phosphor radio buttons) |
| sm | 8px | Top-bar padding (vertical), Debug `<details>` inner padding |
| md | 16px | Top-bar padding (horizontal), wrapper margin between top-bar and canvas |
| lg | 24px | Page-level margin around `#terminal-wrapper` on wide viewports |
| xl | 32px | Unused in Phase 3 — reserved for future phases |
| 2xl | 48px | Unused in Phase 3 |
| 3xl | 64px | Unused in Phase 3 |

**Exceptions (terminal grid geometry — not subject to the 4-px chrome scale):**

| Exception | Value | Rationale |
|-----------|-------|-----------|
| CRT cell size at 1× zoom | 16px CSS wide × 32px CSS tall (bitmap doubled from native 8×16) | Native 8×16 at 1 CSS pixel per bitmap pixel is unreadable on modern laptops (CONTEXT.md "readable at 80×24 on typical laptop without squinting"). 2× upsample makes 80-col fit in ~1280 CSS px and 24 rows fit in 768 CSS px — exactly a 1280×768 viewport region. Integer-multiplier zoom (Ctrl +/−/0) then scales 1×→16×32, 2×→32×64, 3×→48×96, 4×→64×128. DPR multiplication is orthogonal and handled at backing-store level. |
| CRT cell size on backing store | cellCssPx × `devicePixelRatio` | DPR-aware backing store; CSS stays in CSS pixels, canvas internal dimensions stay in device pixels. `imageSmoothingEnabled = false` in CRT theme for pixel-perfect bitmap scaling (PITFALLS #5, SUMMARY §Phase 3). |
| Clean-theme cell size at 1× zoom | 9px CSS wide × 18px CSS tall — measured from JetBrains Mono at 14 px / line-height 1.3 | Validated at planning time by rendering a reference char in an OffscreenCanvas and measuring `ctx.measureText(…).width` + font metrics. Adjustable if the measurement yields non-integer widths (must round to integer CSS pixels for stable 80-col layout). |
| Clean-theme zoom steps | 1× = 14 px font, 2× = 18 px, 3× = 24 px, 4× = 32 px | Integer multipliers are applied to the **cell pixel dimensions**, not the font point size — keeps 80-col geometry stable across zoom steps. Per-zoom-step font sizes listed give JetBrains Mono a legible target without fractional metrics. |
| Bell overlay dimensions | 100% × 100% of canvas | Overlay covers only the canvas, not chrome (D-08). `position: absolute; pointer-events: none;` |
| Scanline overlay dimensions | 100% × 100% of canvas | Same containment as bell overlay (D-11). Only visible in CRT theme. |
| Focus indicator border | 1 px solid (focused) / 1 px transparent (blurred) on `#terminal-wrapper` | Width identical in both states to prevent layout reflow on focus change (D-13). |

**Pre-populated from:** 03-CONTEXT.md D-10, D-13, FEATURES.md "Readable default font at a comfortable default size", SUMMARY.md §Phase 3.

## Typography

**Chrome typography** (top bar, Debug details label, any UI text — not the terminal grid):

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 | 1.4 | Top-bar button labels ("CRT" / "Clean"), phosphor radio labels ("Green" / "Amber" / "White"), Debug `<summary>` label |
| Label | 12px | 400 | 1.3 | Status hints inside the Debug `<details>` pane (e.g., the existing "Paste raw bytes, or use \xNN escapes" hint — retained and restyled) |

**Terminal grid typography** (not chrome — the rendered canvas glyphs):

| Theme | Glyph source | Native size | Rendered size at 1× zoom |
|-------|--------------|-------------|---------------------------|
| CRT | Bitmap 8×16 ROM Uint8Array (D-01) | 8×16 bitmap pixels | 16×32 CSS pixels (2× integer upsample; further multiplied by DPR at backing store) |
| Clean | JetBrains Mono Regular (D-02) | N/A (vector) | Cell ~9×18 CSS pixels, font-size 14px (planning-time measurement confirms exact width) |

**Rules:**

- **Only 2 chrome type roles** (Body + Label) — satisfies "≤ 4 sizes" typography dimension.
- **Only 1 chrome weight** (400 regular) — satisfies "≤ 2 weights" typography dimension.
- Terminal grid glyphs do **not** count against chrome type-role budget; they are a content surface, not a UI typography system. The contract is to keep chrome type roles minimal so the terminal is visually dominant.
- Zoom (Ctrl +/−/0) affects **only the grid glyph rendered size**, never chrome typography. Chrome stays at the fixed Body/Label sizes above at all zoom levels.

**Pre-populated from:** 03-CONTEXT.md D-10 (integer zoom), D-01/D-02 (font sources). Chrome sizes chosen by researcher to match the minimal monospace-chrome aesthetic (D-12).

## Color

Terminal emulator themes (both dark; see FEATURES.md anti-feature row on separate "dark mode") plus chrome surfaces.

### Phosphor palette — CRT theme (D-05 — three user-selectable variants)

| Phosphor variant | Foreground (glyph) | Background (grid) | Accent (cursor block, focus border) | Inspiration |
|------------------|---------------------|-------------------|-------------------------------------|-------------|
| Green (default)  | `#33ff66`           | `#0a0f0a`         | `#33ff66`                           | DEC VT220 P1 phosphor (~525 nm green) |
| Amber            | `#ffb000`           | `#140d00`         | `#ffb000`                           | IBM 5151 / Wyse amber monochrome monitors |
| White            | `#e8e8d8`           | `#0a0a0a`         | `#e8e8d8`                           | IBM MDA P4 phosphor (slightly warm white) |

**Cursor colors in CRT (per phosphor):** block cursor is **inverted phosphor** (bg of the cell behind it swaps to the foreground color; glyph on the cell inverts to the background color). Blurred state: 1 px outlined block in phosphor-fg color, hollow interior (keeps visibility without the inverted-block commitment).

### Clean theme palette (D-04 — single fixed palette)

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0f1419` | Canvas background, `<body>` background |
| Secondary (30%) | `#1e242c` | Top-bar background, Debug `<details>` background, scanline-disabled surface |
| Accent (10%) | `#7fdbca` | **Reserved for: active-theme button highlight, focused cursor block, focus-state 1px border, selected phosphor radio indicator (though phosphor is hidden in clean theme).** |
| Destructive | n/a | No destructive actions exist in Phase 3 (no delete/disconnect/reset in this phase's chrome). Reserved unused. |
| Foreground (glyph) | `#e4e8ee` | Clean theme grid text color |

### Chrome-specific color tokens (shared across themes)

| Token | CRT value | Clean value | Usage |
|-------|-----------|-------------|-------|
| `--chrome-bg` | `#0a0a0a` (near-black, distinct from phosphor bg but harmonious) | `#1e242c` | Top-bar background |
| `--chrome-fg` | phosphor-fg (current phosphor variant) | `#e4e8ee` | Top-bar text color |
| `--chrome-accent` | phosphor-fg | `#7fdbca` | Active button label, focus ring |
| `--chrome-border` | `rgba(255,255,255,0.08)` | `rgba(255,255,255,0.08)` | 1 px top-bar bottom border |
| `--bell-flash` | `rgba(255,255,255,0.7)` | `rgba(255,255,255,0.7)` | CSS variable set on `#bell-overlay` background-color (D-08); same across themes for an obvious flash on both |
| `--scanline-color` | `rgba(0,0,0,0.15)` | unused | Repeating-linear-gradient stripe for CRT scanline overlay (D-11 — ~15% opacity matches CONTEXT.md "subtle enough to use for hours"). |

**Accent reserved for (explicit list — never "all interactive elements"):**
1. The **active theme button label** (the single top-bar button showing the inactive theme's name — "CRT" while clean is active, "Clean" while CRT is active; on hover/focus the accent underlines the label).
2. The **focused cursor block** (via inverted phosphor in CRT, via accent in clean; see Cursor contract below).
3. The **1 px focus-indicator border** on `#terminal-wrapper` (D-13).
4. The **selected phosphor radio indicator** in the CRT phosphor radio group (the 3-way Green / Amber / White selector). In clean theme the radio group is `display:none` (D-12), so this accent never appears there.

Accent is NOT used for: Debug `<details>` summary, hover states on chrome (chrome is static; no hover underline), bell overlay (intentionally white for visibility across phosphor variants).

### Destructive color

No destructive actions exist in Phase 3 chrome. Reserved unused.

## Interaction & State Contracts

The template's six dimensions (Copywriting / Visuals / Color / Typography / Spacing / Registry) don't fully cover a terminal emulator. The following contracts are additive and explicitly called out so the executor implements them deterministically.

### Cursor

| State | CRT theme | Clean theme |
|-------|-----------|-------------|
| Focused + blink-on | Solid block, inverted phosphor (cell-bg becomes phosphor-fg; glyph becomes phosphor-bg) | Solid block in accent color (`#7fdbca`); glyph beneath becomes `#0f1419` |
| Focused + blink-off | Cell renders normally (phosphor-fg glyph on phosphor-bg) | Cell renders normally (`#e4e8ee` glyph on `#0f1419`) |
| Blurred | Steady outlined block: 1 px phosphor-fg border around the cell, cell interior unchanged (no fill, no inversion) | Steady outlined block: 1 px accent border, interior unchanged |
| Blink rate | 530 ms half-period (classic DEC VT cadence — CONTEXT.md specifics); advanced by `frameCount % N` in the rAF render loop — no CSS animation, no separate `setInterval` (D-07) | Same 530 ms cadence |

### Focus indicator

| State | Appearance |
|-------|-----------|
| Focused | `#terminal-wrapper` gets 1 px solid border in theme accent color (CRT: current phosphor-fg; clean: `#7fdbca`) + focused cursor styling above. |
| Blurred | `#terminal-wrapper` gets 1 px solid `transparent` border (same width — no layout reflow) + blurred cursor styling above. |

Both a border change AND a cursor-style change are present — redundant signaling, honors RENDER-03 "border or cursor-style change" with belt-and-braces.

### Bell (BEL 0x07)

| Surface | Behavior |
|---------|----------|
| Screen flash | `#bell-overlay` opacity goes from `0` → `0.7` instantly on BEL, then transitions back to `0` over ~100 ms via CSS `transition: opacity 100ms ease-out`. No canvas damage, no atlas eviction. Overlay is `position: absolute`, covers canvas only, `pointer-events: none` (D-08). |
| Background-tab title | When `document.hidden === true` at the moment BEL fires, set `document.title = "(!) " + originalTitle`. Clear to `originalTitle` on `visibilitychange` → visible. Track with in-flight `hasPendingBell` boolean (D-09). |
| Audible bell | **Not in Phase 3** — v2-AUDIO-01 deferred to at least Phase 6 (CONTEXT.md §deferred). |

### Theme toggle

| Mechanism | Binding |
|-----------|---------|
| Top-bar button | Click the theme button (label shows the destination theme name). |
| Keyboard shortcut | `Ctrl+Shift+T` (D-14; wired via keydown listener on `#terminal-wrapper` with `preventDefault()` so the default Chromium "reopen closed tab" gets recaptured when the terminal has focus). |

### Phosphor selection (CRT theme only)

| Mechanism | Binding |
|-----------|---------|
| Top-bar 3-way radio-group | Segmented control with three buttons (Green / Amber / White); selected item shows accent underline or filled background. |
| Visibility | `display: none` when clean theme active (D-12); no keyboard shortcut (v1 does not add more bindings than ROADMAP requires). |

### Zoom (both themes)

| Binding | Action |
|---------|--------|
| `Ctrl +` | Zoom step up by 1, capped at 4× |
| `Ctrl -` | Zoom step down by 1, floored at 1× |
| `Ctrl 0` | Reset to 1× |

No UI button exists for zoom — keyboard-only per ROADMAP SC-3 and D-10. Zoom change evicts glyph atlas (D-03).

### Debug details

| State | Behavior |
|-------|----------|
| Default (on load) | **Collapsed** (D-15) — page reads as a clean terminal, not a harness. |
| Expanded | Shows the retained Phase 2 textarea, Feed button, and 64 KB Stress button with preserved semantics (the textarea + buttons move into the details pane mostly verbatim from `www/main.js:126-170`). |

## Layout Contract

```
+----------------------------------------------------+
| Top bar  (56 px tall, --chrome-bg, 16px h-padding) |
|                                                    |
|  [theme-button]  [Green|Amber|White]  (← hidden    |
|                                          in clean) |
+----------------------------------------------------+
|                                                    |
|  Canvas area — centered 80×24 terminal grid        |
|  • 1 px focus border on wrapper                    |
|  • Bell overlay (z-index: 1, pointer-events: none) |
|  • Scanlines overlay (CRT only, z-index: 2)        |
|                                                    |
|  Canvas itself: backing store = cellW*80*DPR by    |
|                 cellH*24*DPR; CSS size = cellW*80  |
|                 by cellH*24                        |
|                                                    |
+----------------------------------------------------+
|  <details> Debug  (collapsed by default)           |
|    <summary>Debug</summary>                        |
|    [textarea]                                      |
|    [Feed] [64 KB Stress]                           |
|    hint paragraph                                  |
|  </details>                                        |
+----------------------------------------------------+
```

- Top bar is sticky to the top of the viewport (`position: sticky; top: 0;`).
- Canvas wrapper sits centered horizontally; vertical space above/below the 80×24 grid uses `--chrome-bg` (not the in-theme grid bg) so the grid is visually framed.
- On viewports narrower than the grid at 1× zoom, horizontal scroll is acceptable (daily-driver viewport is assumed to accommodate 80 cols).
- On viewports shorter than the grid, vertical scroll is acceptable.
