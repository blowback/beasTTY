# Milestone v1.1: SLIDE FileTransfer

**Status:** ✅ SHIPPED 2026-05-10

## Overview

The v1.1 FileTransfer milestone adds the SLIDE protocol on top of the v1.0 substrate
so users can move files between PC and MicroBeast without leaving Beastty. The
work was layered exactly along the v1.0 architectural seam:

- **Rust core (wasm)** owns the SLIDE state machine — byte-fed framer,
  CRC-16-CCITT (poly 0x1021, init 0xFFFF, big-endian), sliding-window send /
  receive, RDY / ACK / NAK / CAN / FIN handshakes (SLIDE v0.2 + the v0.2.1
  CAN-bidirectional amendment formalised in ADR-003).
- **JS shell** owns Web Serial bytes, the `ESC ^ S L I D E` wakeup matcher,
  the JS dispatcher that routes inbound bytes to terminal-parser-or-SLIDE,
  the file picker + drag-drop, the floating SLIDE chip, prefs, and the
  Settings rows.

Phase 7 lands the pure-Rust state machine (native `cargo test`, zero browser
deps). Phase 8 adds the wasm boundary, the JS dispatcher, and the wakeup
matcher. Phase 9 delivers host-initiated send (PC → Z80) end-to-end. Phase 10
delivers Z80-initiated receive plus the 5-step ADR-003 cancel sequence and
all the receive edge cases (zero-byte, sub-frame, binary, 1 MB+, idempotent
re-entrant wakeup). Phase 11 wires SLIDE into the v1.0 systems (chip,
prefs, session-log pause, paste-pump gate, port-lost symmetry, echo-swallow,
visibilitychange CTRL_CAN, Compatibility-mode timer). Phase 12 closes the
milestone with collision UX, drag-drop / pointer-select coexistence,
auto-send safety validation, docs (`docs/SLIDE_Z80_REQUIREMENT.md`,
`docs/SLIDE-UAT.md`, README "File transfer" section), and a real-hardware
UAT against a patched MicroBeast.

The v1.1 work spans **2026-05-06 → 2026-05-10** — 5 days end-to-end,
~239 commits, 42 SLIDE-* requirements, all complete.

## Milestone Summary

**Decimal Phases:** None — v1.1 used integer phases only.

**Key Decisions:**

- ADR-003 — SLIDE v0.2.1 CAN-bidirectional amendment formalised: PC-initiated
  CTRL_CAN with Z80 echo, raw single-byte 0x18 wire format (NOT a wrapped
  frame), `force_idle()` escape hatch for stock slide.com that doesn't yet
  support the amendment. Upstream PR target: github.com/blowback/slide.
- 7-byte wakeup signature `ESC ^ S L I D E` (vs 2-byte `ESC ^`) to reduce
  false-positive collisions on benign Z80 output.
- ADR-003 5-step cancel sequence: settle in-flight writes (Promise.allSettled
  ≤ 200 ms) → CTRL_CAN → wait ≤ 500 ms for Z80 echo → drain 100 ms → re-arm
  framer; 2000 ms absolute timeout escape hatch via `force_idle()`. Never
  calls `reader.cancel()` or `port.close()` (keeps terminal session alive).
- Per-file anchor-click download default; `showDirectoryPicker` opt-in
  fallback for batches > 1 file (one user gesture saves all subsequent
  files into the chosen folder); ≥ 250 ms inter-file gap to dodge Chrome
  multi-download throttling.
- Memory-bounded receive: `chunks: Uint8Array[]` + single `new Blob(chunks)`
  at session end (mirrors Phase 6 session-log pattern; avoids O(n²) churn
  for 1 MB+ files).
- Sender filename collision UX: case-insensitive + 8.3 truncation collision
  detection in JS pre-flight; user prompted to auto-rename
  (`NAME.TXT, NAME~1.TXT, NAME~2.TXT`), refuse the batch, or send only the
  first colliding file. `~N` algorithm uses base truncation
  (`Math.max(0, 8 - len(N))`) — supports collision counts up to ~999.
- Auto-send command safety: regex `/^[A-Za-z0-9: ]*\r$/` (admits space for
  default `B:SLIDE R\r`); use-time hard gate at `slide.js
  readAutoSendCommandBytes`; first-use confirmation chip for non-default
  values; rejects `;`, pipes, non-`\r` control characters.
- Compatibility-mode 3-way pref (`auto` / `wakeup-required` / `force-start`)
  for users running pre-v0.2.1 `slide.com` without the wakeup signature;
  `auto` arms a 3 s timer post-auto-type and prompts
  `[Retry] [Cancel] [Force start (legacy slide.com)]` if no wakeup arrives.
- RTS-on-connect prefs gate (`serialAssertRtsOnConnect: true` default) added
  in Plan 12-08 after real-hardware UAT showed RTS-as-reset interfering with
  Z80 connect handshake; Settings checkbox in Connection sub-block.
- Phase 11/12 hard invariant: ZERO Rust changes after Phase 9 — all
  integration-layer work lives in `www/`. Cargo workspace baseline
  preserved at 283 tests across Phases 11+12.
- Diagnostic instrumentation (slideDbg / serialDbg / txDbg) was kept on
  during Phase 12 UAT for hardware bring-up, then stripped in Plan 12-09
  before milestone close (-244 lines net).

**Issues Resolved:**

- Torn-chunk safety across SLIDE frame boundaries (Phase 7 native cargo
  corpus + Phase 8/9/10 Playwright integration tests).
- Idempotent re-entrant wakeup — spurious mid-stream `ESC ^ S L I D E`
  during an active session triggers a clean state-machine reset + chip
  warning ("Z80 reset detected; cancelling current transfer").
- TX writer ownership during sessions: `tx-sink.js setWireOwner('slide')`
  blocks `pushTxBytes` keystroke writes; SLIDE writes via separate
  `writeSlideFrame` path that bypasses the keystroke ring (proves out
  via Playwright tests that keystrokes during a session never corrupt
  the wire).
- Auto-typed-command CP/M echo swallow (~500 ms window) so the typed
  command does not double-print in the terminal.
- Phase 5 port-lost flow extended with `slidePumpOnPortLost` symmetric to
  `pastePumpOnPortLost` — three call sites in serial.js teardown /
  handleReadError / onNavSerialDisconnect.
- Phase 6 session-log gated during active SLIDE sessions so binary frame
  bytes don't pollute the RX log; resumes on session end.
- Phase 6 paste-pump gated during active SLIDE sessions via
  `isSlideActive()` early-return.
- Phase 12 selection / drop-overlay coexistence: 3-line strict-equality
  early-return at top of `selection.js onPointerDown` based on
  `[data-drop-target='true']` attribute; pre-existing v1.0 pointer-select
  behaviour fully preserved.
- UAT-fix session (Plans 12-06..12-09) closed 4 gaps + 3 niggles:
  modal focus-border (data-focused attribute on send-modal footer
  buttons + CSS); autosend invalid-state visual (specificity bump to
  (0,2,2,0) + new --chrome-invalid-strong CSS variable); Force-start
  chip lifecycle (`enterActive()` after `enterSendModeInternal`);
  RTS-on-connect prefs gate (4 connect-time setSignals call sites read
  the pref); diagnostic instrumentation cleanup (-244 lines).

**Issues Deferred:**

- Real-hardware UAT against patched MicroBeast — gated on upstream
  `github.com/blowback/slide` PR merge. UAT scaffold landed at
  `docs/SLIDE-UAT.md` (4-test checklist mirroring the 06/09/10 HUMAN-UAT
  format). UAT-fix session 2026-05-10 closed all blocking gaps; remaining
  hardware sign-off is out-of-band.
- log-download.spec.js filename drift (`bestialitty-` vs `beastty-`) —
  pre-existing from project rename; logged to phase deferred-items per
  scope-boundary rule.
- 10-worker parallel test-suite flakes (slide-recv-settings,
  slide-recv-fsap, slide-cancel timing-window) — green at `--workers=4`;
  cosmetic load-related; logged per phase deferred-items.

**Technical Debt Incurred:**

- None of structural concern. The integration-layer code in `www/` follows
  the v1.0 patterns (modules, dependency injection, Playwright spec
  isolation, mousedown-preventDefault on chrome). The Rust core remains
  pure-logic with the v1.0 boundary contract intact.
- Phase 11 introduced a chip-state introspection API
  (`__getStateForTests`) for test purposes — same convention as v1.0
  scroll-state and tx-sink test introspection.
- Mock-bot grew a sender role (4th independent SLIDE implementation per
  PITFALLS §13 multi-impl drift detection); intentional — it covers the
  Phase 10 receiver E2E without needing a real Z80.
